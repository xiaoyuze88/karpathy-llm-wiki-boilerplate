# lint-wiki

扫描 `wiki/` 目录，检测质量问题，输出诊断报告。

## 执行步骤

1. **扫描所有 wiki/ 文件**
   - 读取 `wiki/index.md` 获取全局视图和页面列表
   - 逐一读取所有子目录下的 .md 文件

2. **检测矛盾**
   - 找出不同页面对同一概念的相互矛盾描述
   - 标记可能过时的具体数据/版本号（创建时间距今 >6 个月）

3. **找出孤立页面**
   - 没有被任何其他 wiki/ 页面以 `[[链接]]` 形式引用的页面

4. **找出缺失概念**
   - 在 ≥3 个页面中被提及但没有专属 `wiki/concepts/` 页面的词汇/概念

5. **检查结构完整性**
   - `wiki/index.md` 中的计数是否与实际文件数一致
   - 各页面 frontmatter 是否完整（title/created/modified/tags/summary 五个必填字段）
   - `wiki/log.md` 中的条目格式是否统一（应为 `## [YYYY-MM-DD] 操作类型 | 描述`）

6. **建议新来源**
   - 根据现有 Wiki 的主题范围和空白区域，推荐 5-10 个值得补充的来源方向
   - 每条建议说明：主题 + 为什么有价值 + 可能的搜索关键词

7. **生成报告**（直接输出到对话，不保存文件）
   ```
   ## Lint 报告 [{日期}]

   ### 矛盾 ({n} 处)
   - 页面A vs 页面B：{描述矛盾内容}

   ### 孤立页面 ({n} 个)
   - [[页面路径]] — {原因或建议}

   ### 缺失概念 ({n} 个)
   - "{概念词}" — 出现在：{页面列表}

   ### 结构问题 ({n} 处)
   - {具体问题描述}

   ### 过时声明 ({n} 处)
   - 页面：{路径}，内容：{原文}，创建时间：{日期}

   ### 建议新来源 ({n} 个)
   - {主题}：{理由} [搜索词：{关键词}]
   ```

8. **追加到 `wiki/log.md`**
   ```
   ## [{日期}] lint | 健康检查
   - 矛盾: {n} 处
   - 孤立页面: {n} 个
   - 缺失概念: {n} 个
   - 结构问题: {n} 处
   - 过时声明: {n} 处
   ```

9. **更新 `VAULT-INDEX.md`**
   - 最近活动追加本次 lint 记录

10. **Git 同步**
    ```bash
    cd ~/knowledge-vault
    git add .
    git commit -m "lint: 健康检查 {日期}"
    git push
    ```

## 完成后输出

完整 Lint 报告 + 优先级建议（哪些问题最值得先修）。
